# #58 ワーク名リネームを「未保存変更」として扱う

- **ステータス**: `[x]` 完了
- **フェーズ**: 14. レビュー対応
- **前提**: #47, #48
- **重要度**: 🟡 UX (U1)

## 概要

`editingWorkNameAtom` の更新は履歴に積まないため、名前のみ変更して戻ろうとした場合 `canUndo === false` で離脱ガード(#48)が発火せず、変更が黙って破棄される。名前変更も「未保存変更」として警告するよう拡張する。

## 受け入れ条件 (AC)

- [x] 名前を変更した状態で戻ろうとすると未保存警告ダイアログが表示される
- [x] 警告ダイアログの選択肢は #48 と同一(キャンセル / 破棄 / 保存して戻る)
- [x] 保存後は警告フラグがクリアされる
- [x] パッチワーク切り替え/読み込み時もフラグがクリアされる

## メモ

- 実装案 A: `editingWorkNameDirtyAtom` を新設し、リネーム時に `true`、保存/読み込み/リセット時に `false`。離脱ガード条件を `canUndo || nameDirty` に拡張。
- 実装案 B: 履歴対象に「名前」を含める(ただし AC #47 では履歴に積まない方針のため避ける)。
- ファイル: `atoms/editor.ts`, `features/editor/EditorScreen.tsx`
