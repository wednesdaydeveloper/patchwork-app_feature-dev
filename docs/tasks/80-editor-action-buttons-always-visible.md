# #80 編集画面の「調整」「対応付けを解除」ボタンを常時表示

- **ステータス**: `[ ]` 実機確認待ち
- **フェーズ**: 仕上げ / UX 改善
- **前提**: なし
- **重要度**: 🟢 UX 改善

## 概要

パッチワーク編集画面で、ピースに布地が割り当てられているか否かによって「調整」「対応付けを解除」ボタンの表示/非表示が切り替わり、その結果アクション行の高さが変動してキャンバスのサイズが微妙に伸縮していた。表示が落ち着かず見苦しいため、これらのボタンを常時表示し、布地未割当時は disabled で見せる方針に変更する。

## 受け入れ条件 (AC)

- [ ] 布地未割当のピース選択時: 「調整」「対応付けを解除」両ボタンが **表示 + disabled** (opacity 0.4)
- [ ] 布地割当済みのピース選択時: 両ボタンが **表示 + enabled** (従来どおり)
- [ ] 調整モード中: 両ボタンは **非表示** (FabricPicker と同じ扱い、従来どおり)
- [ ] ピース選択 → 布地割当 → 解除 の操作でキャンバス領域の高さが変動しない
- [ ] 既存テストグリーン / 型チェック通過

## 変更内容

- ファイル: `features/editor/EditorScreen.tsx` (418〜432行目)
  - 表示条件: `selectedFabricId && !adjustMode` → `!adjustMode`
  - 各 Button に `disabled={!selectedFabricId}` を追加
- `components/ui/Button.tsx` は既に `disabled` prop と `opacity: 0.4` スタイルをサポートしているため変更不要

## メモ

- 案B (調整モード中は両ボタン非表示) を採用。"表示/非表示が切り替わって見苦しい" の対象は「布地未割当 ⇄ 割当」のトグルのみ。
- bug fix のためコミットは実機確認後に行う (CLAUDE.md 方針)。
