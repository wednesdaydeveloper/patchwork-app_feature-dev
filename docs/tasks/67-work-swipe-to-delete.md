# #67 作品一覧のスワイプ削除 UI

- **ステータス**: `[x]` 完了
- **フェーズ**: 14. レビュー対応
- **前提**: #21, #22
- **重要度**: 🟡 UX

## 概要

ホーム画面の保存済み作品一覧で、現在は長押しで削除確認ダイアログを表示している。ユーザーが「長押しで削除」を発見しづらいため、iOS の Mail アプリ等で一般的な「左スワイプで削除ボタン表示」UI に変更する。

## 受け入れ条件 (AC)

- [x] 作品一覧アイテムを左方向にスワイプすると、右側から赤い「削除」ボタンが表示される
- [x] 削除ボタンをタップすると確認ダイアログ(既存)が出て、確定すると DB から削除される
- [x] スワイプを途中で離すと自動で閉じる
- [x] 別のアイテムをスワイプ/タップした際、開いているスワイプは自動で閉じる
- [x] タップは従来通り編集画面遷移
- [x] 長押し削除は廃止(スワイプに統一)

## メモ

- `react-native-gesture-handler` の `Swipeable` コンポーネントを利用(v2.x で同梱)。
- ファイル: `features/home/WorkListItem.tsx`
- 翻訳キー `home.swipeDelete` 等は不要(ボタン文言は既存の `common.delete` 流用)。
- 布地一覧(︙メニュー方式)との UI 不統一は許容(作品=スワイプ、布地=メニュー)。将来的に統一するかは別途検討。
