# #48 未保存変更の離脱警告

- **ステータス**: `[x]` 完了
- **フェーズ**: 13. UX 追加機能
- **前提**: #35, #14

## 概要

未保存の変更がある状態で編集画面を離れようとした場合に確認ダイアログを表示する。

## 受け入れ条件 (AC)

- [x] `canUndoAtom === true`（履歴があり、保存後 clear されていない）状態で戻る／別画面遷移しようとすると確認ダイアログ
- [x] 「破棄して戻る」「保存して戻る」「キャンセル」の 3 択
- [x] 「保存して戻る」は通常の保存フロー（名前未確定なら PromptDialog）後に遷移
- [x] 設定画面・布地管理画面など編集状態を持たない画面では発火しない

## メモ

- Expo Router の `useNavigation().addListener('beforeRemove', ...)` で離脱イベントをフック。
- ダイアログ表示中はナビゲーションをブロックし、ユーザー選択後に解除して再発火する。
- ハードウェアバックボタン（Android）も同じハンドラで扱われる。
