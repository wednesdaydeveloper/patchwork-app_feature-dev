# #54 「保存して戻る」が新規(未命名)ワークでも機能するようにする

- **ステータス**: `[x]` 完了
- **フェーズ**: 14. レビュー対応
- **前提**: #48
- **重要度**: 🔴 不具合 (B1)

## 概要

#48 の離脱ガードで「保存して戻る」を選んだ際、`editingWorkId === null`(新規作成中)の場合は名前入力プロンプトを表示するだけで、保存後の遷移が連動していない。AC「保存して戻るは保存後に遷移」を満たすよう修正する。

## 受け入れ条件 (AC)

- [x] 新規(未保存)ワーク編集中に戻ろうとしてダイアログで「保存して戻る」を押すと、名前入力プロンプトが表示される
- [x] 名前入力後 `保存` を押すと DB に保存され、続けて元の遷移(戻る/別画面遷移)が実行される
- [x] 名前入力プロンプトを `キャンセル` した場合は遷移はキャンセルされ、編集画面に留まる
- [x] 既存ワーク(`editingWorkId !== null`)の場合の動作は変えない

## メモ

- 実装案: 「pending leave action」を `useState` で保持し、`handleSave` 完了時にこれを dispatch して null にクリアする。
- もしくは `handleSave` を引数 `{ name; afterSave?: () => void }` に拡張する。
- ファイル: `features/editor/EditorScreen.tsx`
