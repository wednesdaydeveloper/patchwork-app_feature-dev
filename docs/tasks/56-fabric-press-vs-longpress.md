## 概要

`FabricListItem` でタップ=編集、長押し=削除の二つを同一 `Pressable` にバインドしている。プラットフォームによっては長押し検出後に onPress も発火し、編集ダイアログと削除確認ダイアログが同時に表示される恐れがある。

## 受け入れ条件 (AC)

- [x] iOS / Android 実機で長押し時に onPress が発火しないことを確認
- [x] 発火する場合は `onLongPress` 内で発火フラグを立て、続く `onPress` を抑制する
- [x] 編集と削除の起動が排他である
