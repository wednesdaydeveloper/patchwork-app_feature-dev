# #59 離脱ガードダイアログのボタン順序を改善

- **ステータス**: `[x]` 完了
- **フェーズ**: 14. レビュー対応
- **前提**: #48
- **重要度**: 🟡 UX (U2)

## 概要

現在の離脱ダイアログのアクション順は `[キャンセル, 破棄して戻る, 保存して戻る]`。一般的な UX ガイドラインでは主アクション(保存)を先頭、または右端の最も押しやすい位置に置く。順序を改善する。

## 受け入れ条件 (AC)

- [x] アクション順を `[保存して戻る, 破棄して戻る, キャンセル]` に変更
- [x] iOS / Android 共にプラットフォームの慣習に過度に違反しない
- [x] スクリーンリーダーでも自然な読み上げ順

## メモ

- ファイル: `features/editor/EditorScreen.tsx`
- iOS は通常右が主アクション、Android は左が主アクションという文化差はあるが、本アプリは縦並び実装のため上から重要順で問題なし。
